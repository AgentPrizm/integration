# Security Policy

This repository contains public example integrations. Do not open issues or pull requests containing secrets, API keys, customer data, or confidential prompts.

## Reporting security concerns

For product security concerns, use the contact path listed on the AgentPrizm security page:

https://agentprizm.com/security

## Example safety checklist

- Never commit `.env` files.
- Use demo containers such as `demo-acme`, not production containers.
- Use fake users, fake accounts, fake tickets, and fake sales opportunities.
- Redact personal data before turning a real workflow into an example.
- Add `validUntil` to time-sensitive facts.
