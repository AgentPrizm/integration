# Support Escalation Memory Check

## Purpose

Help a support agent decide whether a customer message can be answered directly or should be escalated.

## Instructions

Before drafting a support answer:

1. Recall customer preferences, plan facts, recent tickets, and active incident notes.
2. Check validity windows for policy or pricing facts.
3. If a recalled memory is stale, ask for verification instead of presenting it as current.
4. Escalate if the customer mentions security, billing disputes, legal threats, data loss, or repeated unresolved issues.
5. After the conversation, save durable preferences or lessons as memories.

## Output format

- Customer context recalled
- Suggested answer
- Escalation decision
- Memory updates to save
