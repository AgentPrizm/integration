# PR Review House Style

## Purpose

Review pull requests against the team's coding and documentation standards.

## Instructions

When reviewing a PR:

1. Recall repo conventions from AgentPrizm before commenting.
2. Check whether the change follows existing naming, security, and test patterns.
3. Flag secrets, credentials, private keys, and customer data immediately.
4. Prefer concise comments with a concrete fix.
5. Summarize repeated feedback as a new `lesson` memory after the review.

## Output format

- Summary
- Blocking issues
- Non-blocking suggestions
- Memories worth saving

## Safety

Never paste secrets or private customer data into a public PR comment.
